# System Design

Коллекция примеров system design для различных систем — от высоконагруженных e-commerce платформ до распределённых сервисов реального времени.

## Цель репозитория

Разбор архитектурных решений на конкретных кейсах: какие компоненты выбраны, какие альтернативы рассматривались и почему отвергнуты, где находятся компромиссы (consistency vs availability, latency vs throughput, простота vs масштабируемость).

Каждый пример включает:
- **Постановку задачи и допущения** — нагрузка, SLA, бизнес-ограничения
- **Высокоуровневую архитектуру** — сервисы, потоки данных, протоколы
- **Обоснование выбора** — почему именно такой стек и паттерны
- **Граничные случаи** — отказы, гонки, восстановление
- **Точки роста** — что меняется при росте нагрузки в 10x/100x

## Структура

Каждая система — отдельная директория с собственным README и диаграммами.

| Пример | Описание |
|---|---|
| [`e-commerce-system`](./e-commerce-system) | Микросервисный e-commerce: Orders, Payments, Inventory, Notifications. Saga, outbox, идемпотентность |

## Стек тем

Разбираемые паттерны и технологии:
- Микросервисы, event-driven архитектуры, saga/outbox
- Синхронное (gRPC, HTTP) и асинхронное (Kafka, RabbitMQ) взаимодействие
- Хранилища: PostgreSQL, Redis, поисковые движки, time-series
- Согласованность, идемпотентность, дедупликация
- Кеширование, шардирование, репликация
- Наблюдаемость: метрики, трейсинг, логирование

## Формат

Документация — на русском, с акцентом на «почему», а не «что». Код-примеры на Node.js (Fastify/NestJS), где это уместно.
